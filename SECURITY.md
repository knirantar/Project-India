# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Project India, please **do not** open a public issue. Instead, please report it responsibly to maintain the security of the project and its users.

### How to Report

1. Email security reports to the project maintainers (create an issue on GitHub and mark it as security-sensitive)
2. Include a clear description of the vulnerability
3. Provide steps to reproduce the issue if possible
4. Allow time for investigation and patching before public disclosure

## Security Considerations

Project India handles research data and analysis. Please note:

- **Data Sensitivity**: Be mindful of sensitive information in datasets
- **Source Verification**: Always verify data sources and validate analysis
- **Database Security**: Use strong credentials for database access
- **API Keys**: Never commit API keys or secrets to the repository

## Best Practices

Contributors should follow these practices:

- Keep dependencies updated
- Report bugs and vulnerabilities responsibly
- Use secure authentication for database connections
- Validate and sanitize data inputs
- Review code carefully before merging

## Scope

This security policy applies to:

- The Project India repository and codebase
- Associated databases and data pipelines
- All research data and analysis tools
- Infrastructure used to run the project

Thank you for helping keep Project India secure.
