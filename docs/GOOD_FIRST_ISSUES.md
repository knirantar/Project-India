# Good First Issues

These tasks are intentionally small and useful for new contributors.

## 1. Improve README Quickstart

Add a concise quickstart that gets a new contributor from clone to local dashboard.

## 2. Add Screenshots

Add dashboard screenshots to the README after the UI has stable example data.

## 3. Add Sample Seed Data

Create tiny synthetic seed rows that help contributors test the Postgres-first flow locally.

## 4. Add Dashboard Filters

Add simple filters for topic category, status, or evidence type.

## 5. Add Database Schema Explanation

Document the major tables in `db/schema.sql` and how they relate to research workflows.

## 6. Add GitHub Actions Badge

Add a CI status badge to `README.md` once the workflow is stable on `main`.

## 7. Add Docs for Local Postgres

Expand setup docs for Docker, local database URLs, initialization, and troubleshooting.

## 8. Add Issue Template Config

Keep `.github/ISSUE_TEMPLATE/config.yml` current as contributor support channels evolve.

## 9. Add Tests for CLI

Add tests for parser construction, command routing, and user-facing CLI output.

## 10. Improve Error Messages

Make database and dashboard errors clearer, especially for missing Postgres, missing dependencies, or empty archive data.
